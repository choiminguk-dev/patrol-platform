## 04-08 작업 (Zone 시스템 + 정정 도구)

### 1) 상습지역 Zone 데이터 모델 + CRUD (Phase A)
- DB 스키마: `patrol_zones` (id, seqNo, address, landmark, notes, referencePhotoUrls[])
- `patrol_entries.zoneId` FK 컬럼 추가
- API: GET/POST/PATCH/DELETE `/api/zones`
- 신규 페이지: `/admin/zones` (관리자 전용 네비게이션 탭)
- ADMIN만 수정/삭제 권한, 안전담당은 읽기 전용

### 2) AI 자동 Zone 매칭 (Phase B)
- analyze-batch에 zone 목록을 시스템 프롬프트로 주입
- AI 응답 스키마에 `zoneSeqNo` 추가
- 후처리: zoneSeqNo → zoneId 매핑 + 주소 폴백 자동 매칭
- TrackB 리뷰 그룹 카드에 zone 드롭다운 (수정 가능)
- AI 추천 시 초록 하이라이트

### 3) Zone 학습 (Phase C-1, C-2)
- 제출 시 zone에 사진 누적 (`referencePhotoUrls`, 최근 20장)
- 다음 분석 시 각 zone의 최근 2장을 320px 썸네일로 prepend
- 시스템 프롬프트에 "참고사진과 시각적 유사 → 같은 zone" 규칙 추가
- 시간이 갈수록 정확도 자동 상승

### 4) 구역 분배 (Redistribute)
- API: `POST /api/entries/[id]/redistribute`
- 사진별 zone 할당 → zone별 새 entry 생성, 원본 삭제
- 공용 컴포넌트 `RedistributeModal` 분리 (Dashboard + TrackB 양쪽 사용)
- TrackB에서는 로컬 state 분할, Dashboard에서는 API 호출

### 5) TrackB 그룹 분할 (Split)
- zone 없이도 그룹을 N개로 단순 분할
- 사진 탭하여 번호 토글 (1→2→3→4→5→1)
- 그룹별 색상 (emerald/blue/amber/pink/violet)
- 신규 컴포넌트: `SplitGroupModal`

### 6) 같은 zoneId 자동 머지 (후처리 4)
- AI가 같은 zone 사진을 여러 그룹으로 쪼갠 경우 자동 합침
- 13건 분석 결과 → 6건으로 깔끔하게 정리
- description은 가장 짧은 것 선택, 첫 photoIndex 기준 정렬

### 7) description prefix 오염 수정
- 기존: 7장 이상 그룹 → description에 "[검토 필요: N장]" prefix 추가
- 분배 후 1-2장 entry에 prefix 잔존 → 잘못 표시
- 수정: prefix → `needsReview` 별도 플래그
- TrackB 그룹 카드에 amber 테두리 + "검토 필요" 배지로 시각화
- redistribute에 `cleanMemo()` 함수 추가 (분배 시 prefix 제거)

### 8) 자체 학습 zone 시스템
- DB: `patrol_zones`에 `category`, `discoveredFrom` 컬럼 추가
- DB: `category_address_stats` 테이블 신규 (카테고리×주소 빈도 누적)
- API: `GET /api/zones/candidates?category=xxx&min=2`
- API: `POST /api/zones/promote` (1-클릭 zone 승격)
- zones 페이지: 카테고리 탭 (전체/상습지역/이면도로) + 후보 구역 섹션 (보라색 강조)
- 사용 흐름:
  1. 사용자가 사진 업로드 + 분배/수정 + 제출
  2. 카테고리×주소 빈도 자동 누적
  3. 빈도 2회+ 도달 → "후보 구역" 자동 표시
  4. 사용자 1-클릭 promote → 정식 zone
  5. 다음 분석부터 자동 매칭

### 9) 주소 정정 학습 (address_corrections)
- DB: `address_corrections` 테이블 (originalAddress → correctedAddress, frequency)
- TrackB: PhotoGroup에 `originalAddressAi` 필드 (AI 첫 제안 보존)
- 제출 시 `addressText !== addressTextAi`면 정정 패턴 누적
- analyze-batch: 정정 패턴 상위 10개를 시스템 프롬프트에 자동 포함
- 흐름:
  1. AI 첫 제안 "두텁바위로55길 2"
  2. 사용자 수정 "두텁바위로 155"
  3. address_corrections에 (두텁바위로55길 2 → 두텁바위로 155) freq=1
  4. 다음 분석에 자동 반영 → 같은 실수 반복 차단

- 사용자가 펼쳐서 AI 판단 검증 가능

### 5) OCR 신뢰 강화 (5차 개선)

#### 5-1. correctAddress() 퍼지매칭 약화
- 두텁바위로XX길 (1~99): KNOWN에 없어도 그대로 신뢰
- 후암로XX길 / 후암로XX가길 (1~99): 그대로 신뢰
- 신흥로XX길 (1~99): 그대로 신뢰
- 100+ 비정상 번호일 때만 fuzzy 매칭
- 일반 fuzzy match (편집거리) 제거

#### 5-2. 시스템 프롬프트 약화
- 기존: "이 외 도로명 존재하지 않음 → 가장 유사한 도로명으로 교정"
- 변경: "참고 목록일 뿐, OCR 결과를 그대로 신뢰"
- 구체 예시 추가: "후암로4길 41" → 그대로 출력
- "강제로 KNOWN에 맞추지 마세요" 강조

이 두 가지를 모두 약화해야 OCR 결과가 그대로 유지됨. 코드 약화만으로는 부족했음 — 프롬프트가 더 큰 영향.

### 6) KakaoMap 마커 렌더링 버그
- 증상: 백엔드 정상(좌표 6건 OK)이나 마커 표시 안 됨
- 원인: SDK 로드(비동기) vs 마커 useEffect 타이밍 문제
  - 첫 마운트 시 mapRef.current가 null → useEffect return
  - 이후 SDK 로드 완료해도 markers prop 변경 없으면 재실행 안 됨
- 수정: `ready` state 추가, 마커 useEffect 의존성에 포함
- SDK 로드 완료 시 `setReady(true)` → 마커 useEffect 자동 재실행

### 7) TrackB 그룹 병합

  - 첫 그룹의 메타 유지 (address/memo/category/zone)

## 정정 도구 3종 비교

| 도구 | 입력 | 결과 | 사용 케이스 |
|------|------|------|------------|
| **그룹 분할** (파란) | 1개 그룹 + 사진별 번호 1~5 | 1 → N개 그룹 | 여러 위치가 한 그룹에 묶임 (zone 무관) |
| **그룹 병합** ★ NEW | 2개+ 그룹 + 체크박스 | N → 1개 그룹 | 같은 위치가 여러 그룹으로 쪼개짐 |
| **구역 분배** (초록) | 1개 그룹 + 사진별 zone | 1 → N개 그룹 (zone별) | zone 등록되어 있고, 한 그룹에 여러 zone 사진 혼재 |

세 도구 조합으로 어떤 잘못된 그룹핑도 한 화면에서 정리 가능.
