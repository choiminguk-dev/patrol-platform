## 0. 한 줄 요약

> "행정 시스템이 아니라 **업무 기록 도구**." 적극행정 시범운영으로 시작해 실사용 데이터를 확보하고, CSAP 인증 국내 클라우드에서 안정적으로 운영하는 PWA.

---

## 1. 포지셔닝 (확정)

```
이 앱 = "업무 기록 도구"
↑↑
"행정 시스템" 아님 — CSAP 국내 클라우드 + 적극행정 범위에서 시범 운영
= 담당자 업무를 줄이는 도구, 결재·인허가 시스템 대체 아님
```

---

## 2. 보안/법적 의사결정 (확정 — 코드 변경 시 준수)

### 클라우드
- ✅ NHN Cloud(현재 사용), Naver, KT, 가비아 — CSAP 인증
- ❌ AWS, Vercel, Railway, Oracle — CSAP 미취득

### 데이터 경계 (Privacy by Design)
```
✅ 앱 저장: 사진, GPS, 카테고리, 메모, 실적
❌ 앱 미저장: 위반자 인적사항, 과태료 공문 원본, 결재 문서
   → 입력 필드 자체가 없음 (설계 단계 원천 차단)
   → 새올행정망에서 처리

✅ Claude API 전송:
   - 사진: 얼굴 블러 후 전송 (face-detect-client.ts + uploads route)
   - 보고서: 텍스트 메타데이터만 (사진 X)
❌ Claude API 미전송:
   - 사진 원본
   - 위반자 정보 (앱에 없음)
```

### 코드 작성 시 체크리스트
- [ ] 위반자 이름/전화/차량번호 입력 필드를 추가하지 않는가?
- [ ] 사진을 API에 보낼 때 블러 처리를 거치는가?
- [ ] 보고서 생성 시 사진 데이터가 아닌 텍스트만 전달하는가?
- [ ] 새로운 외부 API가 CSAP/국내 호스팅을 만족하는가?

## 3. 배포 방식 (확정)

### PWA + 적극행정 공문 (앱스토어 배포 안 함)
```
1. CSAP 인증 국내 서버에 PWA 배포 (https://patrol.ai.kr) ← 완료
2. 적극행정 승인 공문에 URL + QR + 사용 안내
3. 동장 결재 업무 협조 공문으로 사용자 안내
4. "적극행정 시범운영" = 공식 업무 도구 인식
```

### 앱스토어 배제 이유
- 구청 공식 등록: 의사결정 라인 길고 기능 제한 가능성
- 개인 명의 등록: 현직 회색지대
- PWA가 카메라/GPS/오프라인 다 지원

## 4. 데이터 품질관리(QC) 경계

본 저장소는 **동 단위 자가 품질관리(QC)** 기능만 포함한다. 중앙 통합 관측·이상치 탐지 등 상위 QC 기능은 포함하지 않음.

### 본 저장소 범위 (동 단위 자가 QC)
- 구역 상세 조회·편집 (주소·장소특징·설명)
- 구역별 참고 사진 개별 삭제·추가 (담당자가 현장 확인 후 정정)
- 후보 구역 상세 조회 (누적 사진·등록 이력 확인 후 승격·무시 판단)
- 카테고리별 구역 관리 (상습지역 순찰·이면도로 청소·기타 관리)

### 설계 원칙
- 동 단위 자기완결: 본인 동 담당자가 즉시 데이터 정합성 확인·정정
- 오작동 방지: 삭제는 confirm 한 번, 최근 20장만 누적 유지
- 학습 데이터 품질: 부정확한 참고 사진을 삭제하면 다음 AI 매칭 정확도 상승

---

## 5. 코드 작성 시 항상 기억할 것

1. **개인정보는 입력 필드부터 없음** — 위반자 인적사항 절대 추가 금지
2. **사진은 블러 후 전송** — 얼굴 블러 파이프라인 우회 금지
3. **보고서는 텍스트만** — 사진 데이터를 `doc-generator.ts`에 넣지 말 것
4. **CSAP 국내 클라우드 유지** — 외부 API 추가 시 지역/인증 확인
5. **TrackB 철학** — "안 해도 되는 일이 사라지는 것" (담당자가 10분 투자로 끝)
