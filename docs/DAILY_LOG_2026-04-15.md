# 2026-04-15 — 업로드 hang 진짜 근본원인

## TL;DR
5번 연속 엉뚱한 롤백(face-api, 주소판, EXIF 분할)을 유발한 "홈탭 사진 업로드 무반응"의 실제 원인은 **FileList 클리어 레이스**였음. face-api hang이 아니었음.

## 근본원인

```jsx
// 버그 있던 코드 (track-b.tsx, track-a.tsx)
onChange={(e) => { handleUpload(e.target.files); e.target.value = ""; }}

async function handleUpload(files) {
  if (!files?.length) return;
  await acquireWakeLock();       // ← 첫 await
  const arr = Array.from(files); // ← 이미 files는 empty FileList
  // arr.length === 0 → 아무 일도 안 일어남
}
```

- `handleUpload`는 async → 첫 `await`에서 promise 반환
- caller가 동기적으로 `e.target.value = ""` 실행
- **live FileList가 그 자리에서 비워짐**
- `await` 재개 후 `Array.from(files)` → `[]`

콘솔 로그로 명확히 잡힘:
```
[UP-1] onChange fired, files= 1
[UP-4] files size= Array(0)     ← 여기서 0으로 바뀜
```

## 수정

- `src/app/admin/entry/track-b.tsx`: `handleUpload` 진입 직후(첫 await 전) `const arr = Array.from(files)`
- `src/app/admin/entry/track-a.tsx`: `handleGallery` 동일 처리
- 전 구간 try-catch + alert로 hang 대신 에러 가시화
- 단계별 `[UP-n]`/`[A-GAL-n]`/`[A-CAP-n]` 로그 (재발 시 즉시 지점 특정)

## 환경 이슈 (원인 혼동의 배경)

오늘 세션 시작 시 `/admin`이 무한로딩된 건 별개 문제:
