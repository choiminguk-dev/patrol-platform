# 창의행정 추진계획(안)

## 「AI 환경순찰」 순찰 업무 자동화 시스템 구축

---

### ▣ 추진배경

- 환경순찰 후 순찰일지 작성에 **약 2~3시간** 소요 (수작업 기록)
- 현장사진 정리, 위치 기재, 실적 집계 등 **반복적 행정 업무 과다**
- 공무관·지킴이 등 현장 인력 10명의 업무 결과를 카카오톡 3개 그룹으로 수신 → 하루 30~45건 중 대부분 미확인 유실
- 순찰 기록의 정확성·적시성 확보 어려움 (사후 기억 의존)
- 분기별 제출 서류 8종 준비에 **2~3일 소요**, 지연 제출 시 건당 0.5점 감점
- **AI 기술을 활용한 업무 자동화**로 행정 효율화 필요성 대두
- 정부 「디지털플랫폼정부」 구현 전략 및 「공공부문 AI 전면 도입」 국정과제와 연계

#### 5) 데이터 보안 설계 (Privacy by Design) — 5단계 보호

| 단계 | 위치 | 보호 방법 |
|------|------|----------|
| **1단계: 얼굴 감지** | **사용자 기기 (브라우저)** | face-api.js SSD+TinyFaceDetector 이중 감지 → 얼굴 좌표 추출. **해외 전송 0, 비용 0원** |
| **2단계: 온디바이스 블러** | **사용자 기기 (브라우저)** | Canvas 픽셀화 블러 → 미리보기 표시. **서버 전송 전 기기에서 완결** |
| **3단계: 사용자 확인** | **사용자 기기** | 블러 결과 상세 확인(핀치 줌) + 미흡 사진 제외 기능. **본인 확인 후 등록** |
| **4단계: 서버 블러** | **국내 서버 (NHN Cloud)** | 좌표 기반 sharp 블러 → `_blur` 파일 저장. **AI API 호출 0** |
| **5단계: AI 분석** | Claude API | **블러된 사진만 전송** → 주소판 OCR. 데이터 미저장·미학습, 7일 후 삭제 |

| 구분 | 내용 |
|------|------|
| 원본 사진 저장 | CSAP 인증 **국내 서버**에만 보관 (NHN Cloud 국내 리전) |
| 앱 UI 표시 | **블러 버전 기본 표시** (원본은 `?original=true`로만 접근) |
| 다운로드 (일지용) | **블러 버전** ZIP — 보고서/공개 자료용 |
| 다운로드 (공문용) | **원본** — 과태료 증거 등 행정망 내부용 |
| 미수집 데이터 | 위반자 인적사항, 과태료 공문 원본, 결재 문서 — **앱에 입력 필드 자체가 없음** |
| AI 결과물 | **초안만 반환** — 담당자 검토 후 새올에서 정식 공문 작성 |

```
[사용자 기기]                     [NHN 국내 서버]           [Claude 해외]

사진 선택 + 1280px 압축
  ↓
얼굴 감지 (face-api.js, 폰 GPU)  ← 해외 전송 없음
  ↓ 좌표 [{x,y,w,h}]
  ↓
원본 + 좌표 업로드 ──────→ 원본 저장 (국내 only)
                           sharp 블러 → _blur 저장
                           _blur 버전 ──────→ 주소판 OCR
                                              (얼굴 블러됨)
앱 UI ←──────────────────── _blur 기본 서빙
다운로드 (일지) ←───────── _blur ZIP
다운로드 (공문) ←───────── 원본
보고서 생성 ──────────────── 텍스트만 ──→ 순찰일지 초안
위반자 정보 ──────────────── 앱에 없음    기존대로 새올
```

**핵심: 5단계 보호**
1. "얼굴 감지는 사용자 기기에서" → 감지 데이터(좌표)가 해외로 전송되지 않음
2. "블러도 사용자 기기에서" → 서버 전송 전 기기 내에서 완결
3. "사용자가 직접 확인 후 등록" → 블러 미흡 사진 제외 가능, 책임 소재 명확
4. "국내 서버에서 추가 블러" → 외부 AI 호출 없이 국내에서 완결
5. "AI에는 블러된 사진만 전달" → 얼굴 식별 원천 차단, 데이터 7일 후 삭제

#### 6) 기존 업무 방식 대비 보호 수준 비교

| 구분 | 기존 (카카오톡) | AI 환경순찰 |
|------|---------------|------------|
| 얼굴 블러 | **없음** (원본 그대로 전송) | 감지된 얼굴 자동 블러 처리 |
| 사진 저장 위치 | 카카오 서버 + 그룹방 멤버 전원 단말기 | **CSAP 인증 국내 서버 1곳** |
| 접근 가능자 | 퇴직자 포함 그룹방 멤버 전원 | **로그인 인증된 담당자만** |
| 삭제 통제 | 불가능 (개인 단말기에 무기한 저장) | **관리자 삭제 가능** |
| 사진 해상도 | 원본 그대로 (4000px+) | **1280px 압축** (식별력 저하) |

※ 기존 카카오톡 방식에서는 블러 처리 자체가 없으며, 사진이 그룹방 멤버 전원의 개인 단말기에 무기한 저장됨

#### 7) AI API(Claude) 데이터 처리 정책

| 항목 | 내용 |
|------|------|
| 데이터 저장 | **저장하지 않음** — 분석 완료 즉시 폐기 |
| 모델 학습 | **학습에 사용하지 않음** (Anthropic API 이용약관 명시) |
| 전송 데이터 | 블러 처리된 1280px 사진 (원본 카메라 해상도 대비 ~90% 축소) |
| 미감지 얼굴 | 1280px 압축 상태로 식별력 대폭 저하 + API 서버 미저장·미학습 |

#### 8) 운영 지침

- **위반자 적발 사진**(얼굴이 핵심인 사진)은 이 앱에 업로드하지 않음 → 새올에서 별도 처리
- 환경순찰 사진은 **골목·쓰레기·도로 상태** 촬영이 목적 — 사람 얼굴은 부수적 상황
- 얼굴이 포함된 사진은 4단계 보호 체계가 자동 적용됨

[Claude OCR + 그룹핑] 15장씩 청크 분할
  ↓

[후처리 2.5] 7장 이상 → needsReview 플래그
[후처리 2.6] zoneSeqNo → zoneId 매핑 + 주소 폴백
[후처리 3] 동일 주소 연속 병합
[후처리 4] 같은 zoneId 자동 머지

  ├─ AI 판단 근거 펼쳐 보기
  ├─ 🗺 지도 위치 선택 → 주소 자동 입력
  ├─ 그룹 분할 (1→N)
  ├─ 수동/AI 병합 (N→1)
  ├─ 구역 분배 (1→N zone별)
  └─ 주소/카테고리/메모 인라인 편집
  ↓

  ↓

  ↓

  ├─ 상세 편집 (메모/주소/사진 추가·삭제)
  ├─ AI 병합 (같은 구역·주소 자동 정리)

  └─ 캘린더 + 자동 스크롤
  ↓

  ├─ 카테고리별 색상 마커
  ├─ 🔥 히트맵 (빈도 기반)
  ├─ 마커 클릭 → 주소 인라인 수정

  ↓

  ├─ 4종 AI 문서 자동 생성 (서버 점수 사전 계산)

  ├─ 📁 일지용 구조화 다운로드
  └─ TXT / CSV / 복사 내보내기
  ↓
평가 실적 자동+수동 집계
  ├─ 현장평가: 0.5점/건 자동 누적
  ├─ 상습지역/홍보: 자동 (1건+ = 만점)
  ├─ 과태료: 수동 전용 (실제 부과 기준)
  ├─ 경고판/특수사업: 수동 우선
  └─ 분리배출 3파트: 수동, 목표 비례 계산
      ├─ 폐건전지 (5점 / 1,500kg)
      ├─ 폐소형가전 (5점 / 1,500kg)
      └─ 투명페트병 (5점 / 2,000매)
