## 서비스 현황

| 항목 | 상태 |
|------|------|
| 도메인 | https://patrol.ai.kr |

## ★ 핵심 기능: 사진 기반 순찰 자동화 파이프라인

### 전체 흐름 (2단계 에이전트 파이프라인)

```
공무관이 순찰 중 사진 촬영
  패턴: 작업사진 1~5장 → 주소판 1장 (주소판이 마지막)
        ↓
[1. 클라이언트 압축]
  Canvas API → 1280px JPEG 0.7 (~100KB/장)
  원본 EXIF GPS/촬영시간 추출
  150px 썸네일 즉시 생성
        ↓
[2. 청크 업로드]
  10장씩 서버 전송, 진행률 바
        ↓
[3. AI 배치 분석 — 에이전트 1 (분석)]  ★
  15장씩 청크 분할 → Claude Haiku 전송 (30장 제한 해결)
  주소판 OCR (파란 오각형 표지판)
  촬영 패턴 인식: 작업사진 → 주소판 = 1그룹
  후암동 도로명 11개 검증 + 퍼지매칭 교정
  주소 보정: 우암로→후암로, 두턴바위로→두텁바위로
  후처리: 1장 그룹 병합 + 동일주소 병합
  최근 등록 주소 컨텍스트 참고 (DB 조회)
  누락 사진 자동 보정 (프론트엔드 안전장치)
        ↓
[4. 그룹별 리뷰]
  AI가 묶은 그룹 표시 (주소 + 썸네일 + 장수)
  주소/카테고리/메모 수정 가능
        ↓
[5. 일괄 제출]
  그룹 수만큼 DB 엔트리 생성
        ↓
[6. 보고서 자동 생성 — 에이전트 2 (작성)]  ★
  메모(분석 에이전트가 생성)를 읽고 보고서 작성
  참여인원 자동 집계 (카테고리 기반 풀 추론)
```

## AI 분류 (TrackA 실시간)

- Claude Haiku로 첫 사진 자동 분류
- 구체적 패턴: 청소도구(빗자루/쓰레받기) → road_clean (high)
- 주소판만 있는 사진 → road_clean (medium)
- 최근 2주 카테고리 분포 + 주소 패턴 컨텍스트 전달
- 주소 보정: 우암로→후암로, 두턴바위로→두텁바위로
- AI 추천 배지 클릭 → 카테고리 자동 변경

## 서버 관리

### 배포 (코드 업데이트)

```bash
ssh -i C:\Users\user\Downloads\patrol-key.pem ubuntu@133.186.218.27
cd ~/patrol-platform && git pull origin main && npm run build && sudo systemctl restart patrol
```

npm 패키지 추가된 경우:
```bash
cd ~/patrol-platform && git pull origin main && npm install && npm run build && sudo systemctl restart patrol
```

### 서비스 관리

```bash
sudo systemctl status patrol     # 상태 확인
sudo systemctl restart patrol    # 재시작
sudo journalctl -u patrol -f     # 로그 확인
free -h                          # 메모리/스왑 확인
df -h /                          # 디스크 확인
```

### 빌드 멈출 때 (메모리 부족)

```bash
# 스왑 확인
free -h
# 스왑 없으면 추가
sudo fallocate -l 1G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```
