### 사례요약

2025년 제설 시즌에 제설 인프라 관리 시스템을 자체 개발하여 적극행정 제안·실무 운영한 경험을 토대로, 2026년 AI 환경순찰 통합 플랫폼으로 발전. AI 주소판 OCR·사진 자동 분류, 상습지역 자체 학습, 순찰일지 자동 생성, 청소행정평가 100점 자동 집계, 지도 기반 데이터 정합성 검토, 개인정보 자동 비식별화(얼굴/차번호 블러) 등 연간 약 500시간의 반복 업무를 자동화. 개발비 0원(퇴근 후 개인 PC로 자체 개발), 운영비 월 53,000원으로 전국 226개 자치단체에 확산 가능한 오픈소스 모델 구축.

### ① 추진배경

**동 주민센터 환경순찰 업무 현황**

- 청소담당 공무원 1명이 현장 인력 10명(환경공무관 3, 골목청결지킴이 5, 자원관리사 2)을 관리
- 현장 인력은 카카오톡 3개 그룹으로 순찰 사진 전송 → 하루 30~45건 수신
- 담당자는 수신된 현장사진과 주소판 사진을 한 장씩 대조하며 위치를 수기 입력 (약 1~2시간 소요)
- 이후 엑셀/한글로 순찰일지 수작업 작성 (약 1~2시간 추가 소요)
- 분기별 제출 서류 8종 수작업 준비에 2~3일 소요, 지연 시 건당 0.5점 감점
- 카카오톡 메시지의 약 80% 미확인 유실 → 현장 인력 활동 실적 인정률 약 30%

**문제의 본질**

- 현장 인력은 열심히 일하지만 증빙이 유실되어 실적으로 인정받지 못하는 구조적 모순
- 담당자는 매일 2~3시간을 사진 대조·위치 입력·양식 작성 등 반복 업무에 소비
- 이 문제는 전국 226개 기초자치단체의 동 단위에서 동일하게 반복되는 구조적 문제

**정책 추진 계기**

- 2025년 제설 시즌에 제설 인프라 관리 시스템을 자체 개발하여 적극행정 제안 후 실무 운영
- 해당 경험에서 "디지털 도구로 현장 업무를 바꿀 수 있다"는 확신과 함께 개선 과제 도출
- Phase 1의 한계(데이터 주권 문제, 단일 업무 특화, 현장인력 사용 어려움)를 전면 개선한 AI 환경순찰 플랫폼으로 발전

### ③ 추진 중 어려움 및 해결노력

**어려움 1: 현장 인력의 새로운 도구 사용 어려움**

- Phase 1(제설 시스템)에서 현장 인력(고연령층)에게 앱 사용을 안내했으나 실제 활용이 어려웠음
- 스마트폰 조작에 익숙하지 않은 인력이 다수로, 기존 카카오톡 외 추가 도구 사용에 부담

→ 해결: 기존 업무 체계와의 호환성을 유지하여 설계 (Track B)

  · 현장 인력은 기존 카카오톡 보고 방식을 100% 유지
  · 담당자가 퇴근 전 카톡 사진을 시스템에 일괄 등록 (약 10분)
  · 현장 인력에게 추가 교육이나 행동 변경을 요구하지 않는 구조

**어려움 2: 공적 데이터의 보안 및 데이터 주권 문제**

- Phase 1에서 Vercel(해외 서버)을 사용했으나, 업무 데이터의 해외 서버 저장에 대한 데이터 주권 우려 인식
- 순찰 사진에 위반자 얼굴/차량번호가 포함될 수 있어 개인정보 보호 필요성 대두

→ 해결: CSAP 인증 국내 클라우드 전환 + 개인정보 4단계 보호 설계

  · NHN Cloud(CSAP 클라우드 보안 인증 취득) 국내 서버로 전면 전환
  · 위반자 인적사항은 시스템에 입력 자체를 하지 않는 설계로 원천 차단
  · **5단계 얼굴 보호**:
    ① 사용자 기기에서 얼굴 위치 감지 (SSD+Tiny 이중) — 감지 데이터가 해외로 전송되지 않음
    ② 사용자 기기에서 블러 처리 (Canvas 픽셀화) — 서버 전송 전 기기 내 완결
    ③ 사용자가 블러 결과를 직접 확인 후 등록 — 미흡 사진 제외 가능, 책임 소재 명확
    ④ 국내 서버(NHN Cloud)에서 추가 블러 — 외부 AI 호출 없이 국내에서 완결
    ⑤ AI에는 블러 처리된 사진만 전달 — 데이터 미저장·미학습, 7일 후 삭제
  · 비용: 감지 0원, 블러 0원 → 226개 동 확산 시에도 추가 비용 없음
  · **기존 카카오톡 방식 대비 보호 수준 향상**:
    - 카카오톡: 블러 없음, 그룹방 멤버 전원 단말기에 무기한 저장, 퇴직자도 접근 가능
    - 본 시스템: 감지 얼굴 자동 블러, CSAP 국내 서버 1곳 보관, 인증된 담당자만 접근
  · **AI API(Claude) 데이터 미저장·미학습**: 전송 데이터는 분석 즉시 폐기 (Anthropic 이용약관 명시)
  · **운영 지침**: 위반자 적발 사진(얼굴이 핵심)은 이 앱이 아닌 새올에서 별도 처리

**어려움 3: 별도 개발 예산 없이 시스템 구축**

- 외부 용역을 위한 별도 예산 확보가 어려운 상황
- 기존 업무 시간 내 개발 진행이 현실적으로 어려움

→ 해결: 퇴근 후 개인 PC로 자체 개발

  · AI 도구(Claude)를 활용하여 1인 개발 생산성을 극대화
  · 오픈소스 기술 스택 활용으로 라이선스 비용 0원
  · 약 3개월간 설계·개발을 진행하여 시스템 완성 후 NHN Cloud 배포 완료

**어려움 4: AI 주소 인식 정확도 확보**

- 초기 AI OCR이 도로명 표지판을 잘못 읽거나, 유사한 도로명을 혼동하는 문제 발생 (예: "후암로4길 41" → "후암로 41"로 잘못 축약)

→ 해결: 5차에 걸친 정확도 개선 + 자체 학습 시스템 구축

  · 퍼지매칭 약화: OCR 결과를 신뢰하도록 교정 로직 완화
  · AI 프롬프트 개선: 화이트리스트 → 참고 목록으로 변경
  · 자체 학습 3종: 사용자 정정 패턴이 다음 분석에 자동 반영 → 같은 실수 재발 차단
  · 지도 기반 정합성 검토: 히트맵으로 주소 오류 시각적 즉시 발견·수정

### ④ 미담사례

**"일지 쓸 시간에 현장을 한 번 더 돌 수 있게 됐다"**

- 같이 순찰 업무를 담당하는 안전담당 주무관은 기존에 순찰 후 사진 정리와 일지 작성에 매일 2시간 이상을 소요하고 있었음
- AI 환경순찰 시스템 도입 후, 현장에서 사진을 업로드하면 AI가 주소를 자동 인식하고 순찰일지까지 자동 생성되면서 사무실 복귀 후 양식 작성 업무가 크게 줄어듦
- "이전에는 순찰 끝나고 사무실에서 일지 쓰는 시간이 더 길었는데, 지금은 그 시간에 현장을 한 번 더 돌거나 민원 처리를 할 수 있게 됐다"
- 특히 사진과 주소를 한 장씩 대조하며 입력하던 작업이 완전히 사라진 것에 대해 "이게 가장 큰 변화"라고 평가

**"공무원이 직접 만들었다는 게 놀랍다"**
